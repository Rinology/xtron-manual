# 전기자전거 웹 매뉴얼 관리 가이드 (초보사용자용)

이 프로젝트는 누구나 쉽게 새로운 매뉴얼을 추가하고 관리할 수 있도록 설계되었습니다. 
코딩을 잘 모르시더라도 아래 단계만 차근차근 따라 하시면 새로운 내용을 추가하실 수 있습니다.

---

## 🚀 실시간으로 수정 내용 확인하기 (라이브 뷰)

수정한 내용을 내 컴퓨터에서 실시간으로 확인하려면 아래 명령어를 사용하세요.

### 📝 실행 방법:
1. VS Code 하단 터미널(Terminal)창을 엽니다. (단축키: `Ctrl + `` )
2. 아래 명령어를 입력하고 엔터를 누릅니다.
   ```bash
   npm run dev
   ```
3. 터미널에 뜨는 주소(예: `http://localhost:5174/`)를 `Ctrl + 클릭` 하거나 브라우저에 입력하면 실시간 화면이 열립니다.
   * **팁**: 코드를 수정하고 저장(`Ctrl + S`)하면 브라우저 화면이 자동으로 새로고침되어 반영됩니다!

## 1. 새로운 가이드 컴포넌트 추가하기
모든 가이드는 '페달 장착 방법'이나 'LCD 가이드'처럼 화면 슬라이딩 및 사진/확대 기능을 포함한 전용 리액트 페이지(Component) 형식으로 렌더링 됩니다. 단 2단계만 거치면 됩니다.

### 🚀 1단계: 내용물이 담길 컴포넌트 만들기 (`src/components/` 폴더)
1. **새 파일 만들기**: `src/components/` 폴더 안에 새로운 파일을 만듭니다. (파일 이름 끝에 `.jsx`를 꼭 붙이세요! 예: `MyManual.jsx`)
2. **코드 붙여넣기**: 기존에 있는 `src/components/PedalGuide.jsx` 파일의 코드를 복사해서 새 파일에 그대로 붙여넣습니다.
3. **내용 수정하기**: 복사된 코드 안에서 함수 이름(예: `export default function MyManual`)과 본문의 제목, 설명 텍스트, 아이콘만 새 주제에 맞게 수정하여 틀을 완성합니다.

### 🔗 2단계: 웹사이트 메뉴판에 등록하기 (`src/data/guides.jsx` 파일)
1. **파일 위쪽에서 불러오기(Import)**: `src/data/guides.jsx` 파일의 맨 위쪽(import가 모여있는 곳)에 1단계에서 만든 파일을 불러옵니다.
   ```javascript
   import MyManual from '../components/MyManual';
   ```
2. **원하는 카테고리에 내용 추가하기**: 파일 중간에 있는 `categories` 배열을 살펴보고, 넣고 싶은 곳(예: `id: "basic"`)의 `items: [` 안에 아래처럼 새 내용을 적어줍니다. 기존 양식을 복사해서 수정하면 쉽습니다.
   ```javascript
   {
     id: "my-new-manual",            // 영어로 된 고유 이름 (URL이나 고유값으로 쓰임)
     title: "사이드바에 뜰 새 제목",      // 사이드바 메뉴에 표시될 제목
     icon: <Wrench size={18} />,     // 메뉴 옆에 표시될 아이콘
     summary: [                      // 검색 시 보여줄 요약 설명 (따옴표로 묶어 작성)
       "첫 번째 요약 설명",
       "두 번째 요약 설명"
     ],
     placeholderType: "image",
     placeholderText: "안내 이미지 공간",
     customComponent: MyManual       // ★중요★ 1단계에서 만든 컴포넌트 이름 연결!
   }
   ```
이렇게 추가하고 파일을 저장(`Ctrl+S`)하여 다시 GitHub에 `push`하면, 실제 웹사이트 메뉴에도 생기고 클릭 시 방금 만든 상세 페이지가 보입니다!

### 📂 2단계(메뉴 구성 심화) 카테고리 설정
'상황별 해결방법'처럼 가이드가 많을 때, 중분류를 추가하여 더욱 깔끔하게 정리할 수 있습니다. `src/data/guides.jsx` 파일에서 아래와 같은 구조로 작성하세요.

```javascript
{
  id: "troubleshooting",
  title: "상황별 해결방법",
  subCategories: [ // 중분류 추가
    {
      id: "power-issues",
      title: "전원 및 배터리",
      items: [
        { id: "error-01", title: "전원 안켜짐", ... } // 소분류(최종 가이드)
      ]
    },
    {
      id: "noise-issues",
      title: "주행 및 소음",
      items: [ ... ]
    }
  ]
}
```
*   **팁**: 중분류가 필요 없는 카테고리는 기존 방식대로 `items`를 바로 사용하시면 됩니다. 시스템이 자동으로 감지하여 3단계 혹은 2단계로 알아서 렌더링합니다.

## 3. 사진과 GIF(움직이는 사진) 관리 규칙

이미지가 많아져도 헷갈리지 않게 관리하려면 아래 규칙을 따르는 것이 좋습니다.

### 📸 1단계: 사진 파일 이름 짓기 (Naming Convention)
사진을 열어보지 않아도 무슨 내용인지 알 수 있게 **영문 소문자, 숫자, 하이픈(-)**을 사용합니다.
*   **형식**: `카테고리-행위-상세설명-순서.확장자`
*   **예시**:
    *   `pedal-install-r-01.jpg` (오른쪽 페달 장착 1단계)
    *   `battery-check-port.jpg` (배터리 단자 점검)
    *   `tire-pressure-check.jpg` (타이어 공기압 점검)

### 📂 2단계: 폴더 구조 (Folder Structure)
관련 있는 사진끼리 모아서 하위 폴더에 넣습니다. 가이드별로 나누어 관리하는 것이 좋습니다.

```text
public/
└── images/
    ├── common/             # 로고, 아이콘 등 공통 이미지
    └── guides/             # 가이드 전용 이미지 폴더
        ├── pedal/          # 페달 관련 사진
        ├── battery/        # 배터리 관련 사진
        └── tire/           # 타이어 관련 사진
```
*   **팁**: 폴더를 나누면 사진이 많아져도 원하는 사진을 금방 찾을 수 있습니다!

### 🔗 3단계: 코드에서 보여주기
이미지를 불러올 때는 아래처럼 경로를 적어줍니다.
```html
<img src="/images/guides/pedal/파일명.jpg" />
```
*   **팁**: 파일명과 확장자(예: .jpg)까지 정확하게 적어주세요!

## 💡 이것만은 꼭! (주의사항)
*   **콤마(`,`) 조심!**: 항목과 항목 사이에는 반드시 쉼표가 있어야 합니다.
*   **소문자 사용**: 파일 확장자는 반드시 소문자 `.jpg` 여야 합니다. (대문자 `.JPG`는 인식을 못 할 수 있어요!)
*   **괄호 짝맞추기**: `{`를 열었으면 반드시 `}`로 닫혀 있는지 확인해 주세요.

## 3. 자가진단 마법사 봇 질문 추가/수정하기 (관리자용)

메인 화면의 **로봇(자가진단 마법사)**이 묻는 객관식 질문을 변경하거나, 새로운 문제 해결 시나리오를 추가하는 방법입니다.

### 📝 따라하기 단계:
1. **`src/components/TroubleshootingWizard.jsx`** 파일을 엽니다.
2. 파일 위쪽에 있는 `const WIZARD_FLOW = { ... }` 라는 코드 덩어리를 찾습니다. 이 곳이 마법사의 뇌(시나리오)에 해당합니다.
3. 현재 있는 양식을 참고하여 새로운 질문 덩어리(노드)를 추가하거나 기존 글자를 수정할 수 있습니다.

### 🧩 구조 이해하기
마법사의 시나리오는 서로 꼬리를 무는 **객관식 문항 블록**들로 이루어져 있습니다.

```javascript
const WIZARD_FLOW = {
  // 1. 처음 시작할 때 항상 'start' 라는 이름의 블록에서 질문을 던집니다.
  start: {
    question: "어떤 종류의 문제를 겪고 계신가요?",
    options: [
      { label: "전원이 안 켜지거나 배터리 문제", next: "power_sub_1" }, // next: 선택하면 'power_sub_1' 블록으로 이동!
      { label: "자전거가 잘 안 나감 (주행감 저하)", result: "error-tire" } // result: 선택하면 곧바로 "error-tire" 가이드 화면으로 이동!
    ]
  },
  
  // 2. 위에서 next를 타고 온 'power_sub_1' 블록
  power_sub_1: {
    question: "충전기를 연결했을 때 어댑터(충전기)의 LED 색상은 무엇인가요?",
    options: [
      { label: "초록색 (완충 표시) 이지만 전원이 안 켜짐", result: "error-power" } // 최종 가이드로 이동
    ]
  }
};
```

### 💡 주요 핵심 단어:
*   `question` : 사용자에게 물어볼 질문 내용 (예: "브레이크에서 무슨 소리가 나나요?")
*   `options` : 사용자가 클릭할 수 있는 선택지 버튼들(배열 형식)
    *   `label` : 버튼에 적힐 텍스트
    *   `next` : 이 버튼을 누르면 **이어질 다음 질문 블록의 이름** (흐름이 길어질 때 사용)
    *   `result` : 이 버튼을 누르면 **최종 도착할 가이드 문서의 고유 ID** (예: `error-01`, `pedal` 등. 이 ID는 `src/data/guides.jsx`에 등록된 ID와 일치해야 합니다.)

### 📌 예시: "조립 질문" 시나리오 새롭게 이어보기
`start` 메뉴에 "조립을 막 시작했어요" 버튼을 만들고, 상세 질문으로 페달과 안장을 묻는 방식을 추가하고 싶다면 아래처럼 기존 코드에 끼워 넣습니다.

```javascript
const WIZARD_FLOW = {
  start: {
    question: "어떤 종류의 문제를 겪고 계신가요?",
    options: [
      // ... 기존 옵션들 ...
      { label: "새 자전거 조립을 막 시작했어요", next: "assembly_step_1" } // 1. 시작점에 버튼 추가
    ]
  },
  
  // 2. 새 질문 블록 만들기 (맨 밑이나 중간 어딘가에 추가)
  assembly_step_1: {
    question: "어느 부위를 조립하실 차례인가요?",
    options: [
      { label: "페달 장착하기", result: "pedal" },          // 선택 시 바로 가이드 문서로 보냄
      { label: "타이어 바람넣기", result: "error-tire" }  // 선택 시 바로 가이드 문서로 보냄
    ]
  },
  // ... 기타 블록들
};
```

수정을 완료하고 저장(`Ctrl + S`)하면, 마법사 봇이 즉시 새로운 언어와 시나리오로 대화를 시작합니다!
